# Creating a Token

## Step-by-Step Guide

### 1. Connect Wallet
Visit [builders.fun](https://builders.fun) and connect your wallet (MetaMask, OKX Wallet, Trust Wallet, or any WalletConnect-compatible wallet).

### 2. Click "Launch Token"
Navigate to the **Launch Token** page from the top menu.

### 3. Fill Token Details

**Required fields:**
- **Token Name** — The name of your token (max 96 characters)
- **Ticker Symbol** — Short symbol like BTC, DOGE (max 30 characters)
- **Description** — Describe your token
- **Image** — Upload a logo (PNG, JPEG, WEBP, GIF, max 5MB)

**Optional fields:**
- Website, Twitter, Telegram links
- Tag (meme, defi, gaming, ai, nft, social, utility, other)

### 4. Choose Raised Token (Pair)
Select which token your token will be paired with:

| Pair | Graduation Tiers |
|------|-----------------|
| BNB | 6 / 10 / 18 BNB |
| CAKE | 3,000 / 6,000 / 10,000 CAKE |
| USDT | 4,000 / 8,000 / 12,000 USDT |
| USDC | 4,000 / 8,000 / 12,000 USDC |
| USD1 | 4,000 / 8,000 / 12,000 USD1 |
| U | 4,000 / 8,000 / 12,000 U |
| FIST | 8,000 / 16,000 / 24,000 FIST |

The first tier (smallest amount) is for testing only.

### 5. Configure Tax (Optional)
Toggle **Enable Tax** to set up tokenomics:

- **Buy Tax** — Tax rate on purchases (0-10%)
- **Sell Tax** — Tax rate on sales (0-10%)
- **Tax Allocation** — How tax is distributed:
  - **Funds Wallet** — Sent to your wallet for marketing, development, etc.
  - **Burn** — Permanently burned (deflationary)
  - **Liquidity** — Added back to PancakeSwap liquidity
  - **Dividend** — Distributed to token holders

Tax is only active **after graduation**. During the bonding phase, no tax is applied.

### 6. Initial Buy (Optional)
You can optionally buy tokens in the same transaction as creation. This shows commitment as the creator.

### 7. Create Token
Click **Create Token** and confirm the transaction. Your token will be live and tradeable immediately.

**Cost:** 0.0001 BNB creation fee + gas (~$0.01 total)

## Rate Limit
There is a 1-minute cooldown between token deployments per wallet to prevent spam.
